PROJECT 5: AUTONOMOUS F1 TELEMETRY & FANTASY PREDICTOR
===========================================================================
Title: Autonomous F1 Telemetry & Fantasy Predictor
Tagline: Multi-Agent System for Racing Analytics & Prediction
Technical Stack: Python, AI Agents (CrewAI & LangChain), RAG (LlamaIndex, ChromaDB), FastF1 API, LLMs (Gemini), Helicone, Zapier, Notion

System Architecture & Data Flow:
- Orchestrator Agent: Triggered asynchronously post-qualifying to manage task delegation and state among sub-agents.
- Retrieval Sub-Agent (Telemetry): Executes Python to query the FastF1 API for raw JSON datasets covering lap times, tire degradation, and micro-sector splits.
- RAG Pipeline: Connects to a local ChromaDB instance populated with vectorized historical track characteristics, weather probabilities, and historical safety car rates via LlamaIndex.
- Evaluation Sub-Agent: Synthesizes quantitative FastF1 data with qualitative RAG context to compute projected points.
- Observability Layer: Utilizes Helicone to capture telemetry on LLM calls, monitoring token latency and cost.
- Data Flow: Cron Job Trigger → Orchestrator → Parallel API scrape (FastF1) & Vector Search (ChromaDB) → LLM Synthesis (Gemini) → Output JSON → Zapier Webhook → Notion Database.

Description:
Engineered a headless multi-agent system to automate Formula 1 fantasy predictions by integrating real-time telemetry with historical race context. The pipeline orchestrates parallel data ingestion—scraping the FastF1 API while querying a ChromaDB vector store—before synthesizing the data via LLM to output optimized roster predictions to a structured Notion dashboard for human review. 

Key Metrics & Impact:
- Deployed a highly technical multi-agent architecture to solve time-sensitive data analysis prior to race start.
- Minimized infrastructure costs by utilizing open-source vector databases (ChromaDB), orchestration frameworks (CrewAI), and free-tier LLM integrations.
- Implemented observability tools and logging to prevent silent failures during asynchronous execution loops.

GitHub: https://github.com/jeswellll/f1-telemetry

PROJECT 6: CUSTOM UIUC DASHBOARD (API Integration & Automation)
===========================================================================
Title: UIUC Workspace Dashboard
Tagline: Consolidated Academic, Shift Schedule & Campus News Command Center
Technical Stack: Next.js (App Router), React, TypeScript, Google Calendar API (googleapis), Canvas LMS iCal, Reddit API, RSS Parser, Google Gemini API (gemini-2.5-flash), Tailwind CSS

Description:
Engineered a secure, custom personal dashboard that consolidates academic workflows, work shifts, and campus news into a unified interface. Built direct integrations with the Google Calendar API via JWT service account authentication to stream upcoming work schedules, alongside a Canvas LMS iCal parser to track upcoming assignment deadlines. Leveraged the Google Gemini API to ingest and synthesize daily headlines from the campus newspaper RSS feed and top daily posts from the r/UIUC subreddit, dynamically generating structured, high-priority campus safety alerts and top stories.

Key Metrics & Impact:
- Centralized 4 distinct external data streams (Canvas LMS, Google Calendar, Daily Illini RSS, and r/UIUC Subreddit) into a single asynchronous, low-latency UI.
- Automated daily campus briefing generation using LLM semantic parsing to extract critical safety warnings and top news stories in real time.
- Implemented modular, component-based layout design ensuring instant performance rendering.
